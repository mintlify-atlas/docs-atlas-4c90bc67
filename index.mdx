GTA:SA Collision is a reverse-engineered C++ implementation of the collision detection subsystem used in Grand Theft Auto: San Andreas. It documents the full hierarchy of collision primitives, model management classes, physical simulation, and spatial storage systems that power object interactions in the game engine.

## System overview

<Steps>
  <Step title="Collision primitives">
    Every collidable surface is described by one of four primitive types: `CColSphere`, `CColBox`, `CColLine`, or `CColTriangle`. Each primitive carries a `ColData` struct storing surface type, piece type, and lighting.
  </Step>
  <Step title="Collision models">
    Primitives are collected into a `CCollisionData` object, which is owned by a `CColModel`. A `CColModel` also holds a `CBoundingBox` for broad-phase culling before testing individual primitives.
  </Step>
  <Step title="Detection methods">
    The static `CCollision` class provides all test and process functions. Test functions return a bool; process functions additionally compute the `CColPoint` (position, normal, surface metadata, depth) and a collision ratio.
  </Step>
  <Step title="Physical simulation">
    `CPhysical` extends `CEntity` with mass, velocity, friction, and elasticity. It calls into `CCollision` during `ProcessCollision()` and applies impulses via `ApplyCollision()` and `ApplySpringCollision()`.
  </Step>
  <Step title="Streaming">
    `CColStore` manages named `.col` file slots backed by a quad-tree. It loads collision data on demand as the player moves through the world. `CColAccel` caches collision models to speed up repeated loads.
  </Step>
</Steps>
